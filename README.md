##imitation logo
<構成要素>
緑とオレンジの縞模様、黒と白色のサイズ違いの楕円をそれぞれ2つずつ、TULLY’S COFFEEの下の黒い長方形、TULLY’S COFFEEの文字、白い細い線の長方形3個の構成要素に分割して制作した。
要素1
fill関数で色を設定し、rect関数で縞模様を制作した。for文を用いて、オレンジと緑が交互に来るようにした。
要素2
楕円をelipse関数で描き、中を縞模様にしたかったため、noFill関数で空白にし、strokeWeight関数で枠の幅を決めた。また、はみ出してしまった縞模様を白色の中が空洞の楕円で隠した。
要素3
黒いラベルをrect関数で白色の細い線もrect関数で描いた。Coffeeにまで白色の線が貫通しないように、黒い長方形で背景を描いた。
<制作の際に工夫した点>
TuLLY’S coffeeの文字の大きさを工夫した。TとYと’だけ大きいのでtextSizeを100に、ULLとSは少し小さめの70にcoffeeはさらに小さい40にした。
黒いラベルが楕円の上に貼られているデザインとなっているため、大きめに長方形を描いた。
最初は縞模様の大きさを楕円に合わせようと工夫したが、なかなか上手くいかなかったが、外側の楕円で上手く隠せた。
文字の細かい位置や色の調整など、なるべく本物のロゴに似せることが出来るように工夫した。
<出力されたイラストを元に考察>
元のロゴに比べると、文字のフォントがまっすぐでないが、元のロゴに比べると色合いや楕円の大きさ、ラベルの大きさ、線の太さなどは本物に近づけられたと思う。

##Kirby
<何を作ったか>
星のカービィをモチーフに、敵を吸い込む技を再現しました。動くボールにカービィがぴったりと重なると星が出現するようにしました。カービィがマウスの動きに反応するようなインタラクティブな演出を加えています。
<構成要素>
要素1：キャラクター（カービィ）の描画
関数名： drawKirby(int r)
マウスの座標（mouseX, mouseY）を中心に、ピンク色の円でカービィの体を描いています。目・頬・口などもellipse()やcurveVertex()を使って表現しました。星を出しているときにはキャラクターを少し大きくし、口の開いた演出を加えるなど、インタラクティブな演出を実現しています。
要素2：動く敵キャラクター（ボール）の描画
関数名： drawEnemyBall(int r)
左から右に移動する緑色の円を敵として描画し、x3にspeedを加算して動かしています。マウスのx座標と敵が近づくと速度が上がり、ぴったり重なると星が発射されるようになっています。また、画面の端に達したらリセットして再出現するようにしています。
要素3：星の描画と動き
関数名： drawStar(int n)
5つの頂点を持つ五芒星を描くため、cosとsinを使って外側・内側の頂点を交互に配置しています。星はカービィの口から発射されるように、マウス座標を中心に表示し、横方向に動きながら画面の端で跳ね返る動きを取り入れています。色はランダムにして、より華やかな印象を与えるよう工夫しました。
要素4：ヒット表示と演出
関数名： showHitText(), drawOpenMouth()
星が出た瞬間に「HIT!」という文字が表示され、カービィの口も楕円で大きく開くように描画することで、技を使った瞬間のインパクトを強調しました。
要素5：アニメーション全体の制御
関数名： draw()
draw()内で背景の描画、キャラクター・敵・星などを呼び出しています。星の表示はdoDrawとframeCountを用いて、一定時間のみ出現するように管理しています。
<制作の際に工夫した点>
•	星のカービィをモチーフに、敵と接触したときに星を発射するというアニメの演出を再現しました。
•	カービィの位置に星が出るようにして、「吸い込んで吐き出す」ような動きに見えるように工夫しました。
•	星は五芒星になるように三角関数で頂点を計算し、交互に外側・内側を使って形を整えました。
•	星の表示にはframeCountを使い、一定時間後に自動で消えるようにしてアニメーションの流れが自然になるようにしました。
•	敵との接触で「HIT!」と表示したり、カービィの口を開いたりすることで、視覚的な演出も強化しました。
•	星の色をランダムにすることで、毎回違う印象になり、飽きないアニメーションになるように工夫しました。
•	コードは関数ごとに役割を分けて整理し、インデントやコメントも整えることで可読性を高めました。

##elevator
＜クラスの構成>
•	Object クラス:
People と Elevator の共通の基底クラスです。
すべてのオブジェクトが持つべき共通のプロパティ（位置を示す x と y、行き先を意味する n、上昇中か下降中かを示す isRaised）を管理しています。
stop() メソッドで、オブジェクト（エレベーターや人）の昇降を制御します。
•	People クラス:
Object クラスを継承しています。
画面の右端から現れ、左へ移動する「人」を表現します。
自分の行き先（n=2 は2階、n=3 は3階）に応じて、対応するエレベーターに向かいます。
エレベーターに到着すると、エレベーターの動きに合わせて一緒に昇降します。
•	Elevator クラス:
Object クラスを継承しています。
ビル内の「エレベーター」を表現します。
isEmpty というブール値で、エレベーターが空か、人が乗っているかを管理しています。
人が乗ると isEmpty が false になり、色が灰色に変わります。
•	People2 クラス:
ArrayList<People> を継承しています。
複数の People オブジェクトを管理するためのクラスです。
新しい人をランダムに行き先を決めて生成し、リストに追加する機能を持っています。
<全体の処理の流れ>
1.	初期設定 (setup()):
画面サイズを600x600に設定します。
2つの Elevator オブジェクトを異なる位置に作成します。
People2 オブジェクトを1つ作成し、人の管理を始めます。
2.	描画ループ (draw()):
毎フレーム、背景を白く塗りつぶします。
1階、2階、3階の床となる線を引きます。
2つのエレベーターと、People2 クラスが管理する全ての人を画面に描画します。
エレベーターの昇降 (stop()) や、人の移動とエレベーターとの相互作用 (show()) が行われます。
<操作方法>
•	人の追加:
キーボードの a キーを押すと、画面右端から新しい人が現れます。
この人は、ランダムに2階か3階に行きたいと設定されます。
<アピールポイント>
•	オブジェクト間の相互作用: People クラスが Elevator クラスのインスタンスを参照し、その状態に応じて自身の振る舞いを変化させています。これにより、現実世界のエレベーターと乗客の関係性をシンプルに表現しています。
•	ワールド座標とローカル座標: translate と pushMatrix/popMatrix を使用することで、各オブジェクトの描画を独立したローカル座標系で行っています。これにより、コードがより直感的で、オブジェクトの位置変更が容易になっています。
