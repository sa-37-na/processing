プログラミング作品ポートフォリオ
1. Tully's Coffee ロゴ模写
作品概要
Processing言語を使用し、Tully's Coffeeの企業ロゴを視覚的に再現しました。 幾何学的要素の配置とプログラミングによる描画技術を駆使した作品です。
技術的実装
使用した主要な構成要素:
要素1: 緑とオレンジの縞模様 - fill関数で色を設定し、 rect関数とfor文を用いて交互のパターンを生成
要素2: サイズ違いの楕円 - ellipse関数と noFill関数、strokeWeight関数を活用し、 縞模様が楕円内に収まるよう白色の外枠で調整
要素3: テキスト要素 - rect関数で黒いラベルと 白い細線を描画し、視覚的な階層構造を実現
工夫した点
テキストサイズの階層化: "TULLY'S"の各文字を異なるサイズ(100pt, 70pt, 40pt)で設定し、 オリジナルのデザインを忠実に再現
レイヤー構造の実装: 黒いラベルを楕円の上に配置するデザインを、 描画順序の最適化により表現
縞模様の制御: 楕円との組み合わせにおいて、外側の楕円で不要な部分を効果的にマスキング
色彩と比率の精密な調整により、企業ロゴとしての完成度を追求
参考: TULLY'S COFFEE公式ロゴマーク
2. 星のカービィ - インタラクティブアニメーション
作品概要
人気ゲームキャラクター「星のカービィ」をモチーフに、 マウス操作に反応するインタラクティブなアニメーションを制作しました。 敵キャラクターとの接触判定や星の発射など、ゲームの主要な動作を再現しています。
実装した機能
1. キャラクター描画システム (drawKirby関数)
マウス座標(mouseX, mouseY)に追従するカービィの描画
ellipse関数とcurveVertex関数による キャラクター造形
状態に応じた表情変化(通常時と星発射時で異なる口の開き方)
2. 敵キャラクターの動的制御 (drawEnemyBall関数)
左から右へ移動する緑色の敵キャラクター
マウス位置に応じた速度変化のアルゴリズム実装
画面端での自動リセット機能
カービィとの接触判定による星発射トリガー
3. 星の生成と動作制御 (drawStar関数)
三角関数(cos, sin)を用いた五芒星の幾何学的生成
カービィの口から発射される視覚効果の実装
画面端での反射動作
ランダムな色彩変化による視覚的多様性
4. ユーザーフィードバック機能
接触成功時の"HIT!"テキスト表示 (showHitText関数)
カービィの口が開くアニメーション (drawOpenMouth関数)
5. アニメーション管理システム
draw関数による全体制御
frameCountを活用した時限表示機能
技術的な工夫
三角関数を用いた五芒星の正確な頂点計算により、美しい星の形状を実現
frameCountによるタイミング制御で、自然なアニメーションの流れを構築
ランダム要素の導入により、繰り返しプレイでも飽きない設計
関数の役割分担による可読性の高いコード構造
適切なインデントとコメントによる保守性の向上
参考: 星のカービィポータル - コピー能力
3. エレベーターシミュレーション
作品概要
オブジェクト指向プログラミングの概念を活用し、 3階建てビルのエレベーターシステムをシミュレートしました。 複数のオブジェクト間の相互作用を視覚的に表現した作品です。
クラス設計
1. Objectクラス(基底クラス)
PeopleクラスとElevatorクラスの共通基盤
位置情報(x, y)、行き先(n)、昇降状態(isRaised)を管理
stop関数による昇降動作の制御
2. Peopleクラス(継承)
画面右端から出現し、左方向に移動する人の動作を実装
行き先階(2階または3階)に応じた適切なエレベーター選択
エレベーター到着後の同期移動機能
3. Elevatorクラス(継承)
ビル内のエレベーターを表現
isEmptyフラグによる乗車状態の管理
乗車時の視覚的フィードバック(色の変化)
4. People2クラス(ArrayList継承)
複数のPeopleオブジェクトを一元管理
ランダムな行き先設定による動的な人の生成機能
システムフロー
初期設定 (setup関数):
600×600ピクセルのキャンバス設定
2台のエレベーターオブジェクトの配置
人管理システム(People2)の初期化
描画ループ (draw関数):
背景の更新と各階の床の描画
エレベーターと人の状態更新・描画
相互作用の処理実行
ユーザー操作:
キーボード'a'キー押下で新しい人を追加
行き先はランダムに2階または3階に設定
技術的特徴
オブジェクト間の相互作用: Peopleクラスが複数のElevatorインスタンスを参照し、 状態に応じて動作を変化させることで、現実世界のエレベーターシステムを再現
座標系の活用: translate関数と pushMatrix/popMatrix関数により、 ワールド座標とローカル座標を使い分け、各オブジェクトの独立した描画を実現
継承の活用: 基底クラスで共通機能を定義し、 派生クラスで固有の振る舞いを実装するオブジェクト指向設計
状態管理: booleanフラグによる明確な状態遷移の実装
技術スタック
言語: Processing (Java派生)
主要技術: オブジェクト指向プログラミング、イベント駆動、アニメーション制御
数学的知識: 三角関数、幾何学、座標変換
まとめ
本ポートフォリオでは、グラフィックス描画の基礎から、 インタラクティブなアニメーション、オブジェクト指向設計まで、 段階的に高度な技術を習得してきた過程を示しています。 各プロジェクトにおいて、視覚的な完成度とコードの保守性の両立を重視し、 実装を行いました
